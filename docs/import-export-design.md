# 导入导出功能设计文档

## 概述

导入导出功能从依赖服务端 gRPC 改为**纯客户端本地实现**，提高可靠性和离线可用性。

---

## 一、数据导出

### 1.1 导出格式

当前仅支持 **CSV 格式**本地导出。Excel/PDF 需要依赖服务端（暂不实现）。

### 1.2 分类筛选改进

**当前问题**: 一二级分类平铺在一个 Wrap 里，用户难以区分层级。

**改进方案**: 改为 ExpansionTile 树形结构：
- 一级分类作为可展开的 section header
- 二级分类作为 section 内的 checkbox
- 选中一级分类 = 选中该分类下所有子分类
- 支持"全部"快捷按钮

### 1.3 导出按钮修复

**当前问题**: 导出按钮先尝试 gRPC 导出，离线时 fallback 到本地但 Excel/PDF 会报错。

**改进方案**: 
- 移除 gRPC 导出依赖，全部走本地生成
- 只保留 CSV 格式选项（移除 Excel/PDF 选项直到有本地实现）
- 导出后直接调用 share_plus 分享

## 二、数据导入

### 2.1 支持的格式

| 格式 | 编码 | 头部跳过行 | 关键列 |
|------|------|-----------|--------|
| 支付宝 | GBK | 前 4 行 + 最后 7 行 | 交易时间, 商品名称, 金额(元), 收/支, 交易状态 |
| 微信 | UTF-8 BOM | 前 16 行 | 交易时间, 交易类型, 商品, 金额(元), 收/支, 当前状态 |
| 通用 CSV | UTF-8/GBK | 无 | 用户手动映射 |

### 2.2 支付宝账单格式

```csv
# 前 4 行是描述信息（支付宝交易记录明细查询、起始日期、终止日期、空行）
交易号                  ,商家订单号               ,交易创建时间              ,付款时间                ,最近修改时间              ,交易来源地     ,类型              ,交易对方            ,商品名称                ,金额（元）   ,收/支     ,交易状态    ,服务费（元） ,成功退款（元）  ,备注                  ,资金状态       ,
...
# 最后 7 行是汇总信息
```

**解析策略**:
- 检测文件前几个字节判断 GBK 编码
- 跳过前 4 行描述信息
- 截断最后 7 行汇总信息
- 列名映射：`交易创建时间` → date, `商品名称` → note, `金额（元）` → amount, `收/支` → type
- 过滤 `交易状态` != "交易成功" 的行
- `收/支` 列：`支出` → expense, `收入` → income, 其他(如空/不计收支) → 跳过

### 2.3 微信账单格式

```csv
# 前 16 行是描述信息（微信支付账单明细、起始/结束时间、统计信息等）
交易时间,交易类型,交易对方,商品,收/支,金额(元),支付方式,当前状态,交易单号,商户单号,备注
...
```

**解析策略**:
- 检测 UTF-8 BOM (EF BB BF)
- 跳过前 16 行描述信息
- 列名映射：`交易时间` → date, `商品` → note, `金额(元)` → amount, `收/支` → type
- 金额列带 ¥ 前缀，需要去掉
- 过滤 `当前状态` != "支付成功"/"已收钱"/"已存入" 的行
- `收/支` 列：`支出` → expense, `收入` → income, `/` → 跳过

### 2.4 导入流程

```
选择文件 → 自动检测格式 → 解析预览 → 确认导入
     ↓           ↓              ↓           ↓
  file_picker  编码/头部     DataTable    批量写入DB
               检测         + 行数统计    + 去重检查
```

### 2.5 智能分类匹配

根据商品名称/备注中的关键词自动匹配分类：

| 关键词 | 一级分类 | 二级分类 |
|--------|---------|---------|
| 美团, 饿了么, 麦当劳, 肯德基, 星巴克 | 餐饮 | - |
| 早餐, 早点 | 餐饮 | 早餐 |
| 午餐, 午饭 | 餐饮 | 午餐 |
| 滴滴, 出租车, 打车 | 交通 | 打车 |
| 地铁, 公交, 一卡通 | 交通 | 地铁公交 |
| 加油, 中石油, 中石化 | 交通 | 加油 |
| 淘宝, 京东, 拼多多, 天猫 | 购物 | - |
| 水电, 燃气, 物业 | 居住 | 水电燃气 |
| 房租 | 居住 | 房租 |
| 转账, 红包 | 不导入（跳过） | - |

未匹配的默认放入"其他"分类。

### 2.6 去重策略

同一天 + 相同金额 + 相同备注 → 视为重复，跳过导入。
